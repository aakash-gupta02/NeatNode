# Quickstart

Spin up a new Node.js backend using **JavaScript or TypeScript** with NeatNode â€” no setup required.

---

## Create a New Project

Run the CLI command:

```bash
npx neatnode
````

Follow the interactive prompts to choose:

* **Project folder name**
* **Language** (JavaScript or TypeScript)
* **Template type** (Basic, REST API, or Socket.IO)
* **Include example CRUD?** (Yes / No)

---

## After Scaffolding

Once generation completes, enter your project and start the dev server:

```bash
cd <project-name>
npm install
npm run dev
```

> [!TIP]
> Use `npm start` for production mode (if available in the template).

---

## Template Options

### JavaScript Templates

| Template           | Description                                             |
| ------------------ | ------------------------------------------------------- |
| **Basic (JS)**     | Minimal Express starter for learning or small services. |
| **REST API (JS)**  | Express API with CRUD, authentication, and validation.  |
| **Socket.IO (JS)** | Real-time server using Express + Socket.IO.             |

### TypeScript Templates

| Template       | Description                                                                       |
| -------------- | --------------------------------------------------------------------------------- |
| **Basic (TS)** | Minimal Express setup with TypeScript, strict typing, and preconfigured tsconfig. |
| REST API (TS)  | *(Coming soon)*                                                                   |
| Socket.IO (TS) | *(Coming soon)*                                                                   |

## Example Workflow (TypeScript)

```bash
npx neatnode
# Enter project folder name: my-ts-app
# Select language: TypeScript
# Choose template: Basic Express (TS)
# Include CRUD: No

cd my-ts-app
npm install
npm run dev
```

Server starts at:

```text
http://localhost:3000
```
