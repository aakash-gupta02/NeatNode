# Folder Structure

Templates produced by the CLI share an MVC-like layout. Example layout produced by the templates:

```
<project-root>/
├─ package.json
├─ server.js
├─ .env.example
└─ src/
	├─ app.js              # Express app: middleware and route wiring
	├─ config/
	│  ├─ env.config.js
	│  ├─ db.config.js
	│  └─ logger.config.js (REST / Socket templates)
	├─ controllers/        # Request handlers
	├─ models/             # Mongoose models (if used)
	├─ routes/             # Route definitions
	├─ services/           # Business logic layer (REST template)
	├─ middleware/         # Auth, error handling, rate limiter
	└─ utils/              # Helpers (ApiResponse, ApiError, Token, CatchAsync)
```

Notes:

- The Basic template includes an optional `todo` CRUD set (models, controllers, routes). If you decline CRUD during scaffolding, these files are removed.
- The REST template contains authentication, validation, and a service layer. The Socket template wires Socket.IO with an HTTP server and socket initializer.
