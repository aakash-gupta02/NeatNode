# FAQs

Common questions developers have about using **NeatNode**, its CLI, and its generated templates.

---

## General

<details>
  <summary><strong>How do I create a new project with NeatNode?</strong></summary>

Run the CLI directly using **npx**:

```bash
npx neatnode
````

You‚Äôll be prompted for:

* **Project name**
* **Language** (JavaScript or TypeScript)
* **Template type** (Basic, REST API, or Socket.IO)
* **Whether to include CRUD modules**

After generation:

```bash
cd my-app
npm install
npm run dev
```

</details>

---

<details>
  <summary><strong>How do I change the default template?</strong></summary>

You can choose a template during the CLI prompt, or edit the generated files manually later.
If you want to make a new default, open **`src/config/templates.js`** and reorder or modify the entries.

</details>

---

<details>
  <summary><strong>How can I add my own custom template?</strong></summary>

1. Create a new folder under `backend/templates/` (e.g., `express-graphql`).
2. Add all necessary files: `server.js`, `src/app.js`, `package.json`, etc.
3. Register it in **`templates.js`**:

   ```js
   { name: "GraphQL API", path: path.join(__dirname, "../../templates/express-graphql") }
   ```
4. Run `npx neatnode` and select your new template from the list.

</details>

---

<details>
  <summary><strong>How do CRUD files get removed?</strong></summary>

NeatNode removes CRUD-related files and route references using markers in `app.js`:

```js
// ROUTE_IMPORTS_START
// ROUTE_IMPORTS_END

// ROUTE_USES_START
// ROUTE_USES_END
```

When you select ‚ÄúNo CRUD‚Äù during setup, these files are deleted:

* `src/models/*.model.js`
* `src/controllers/*.controller.js`
* `src/routes/*.route.js`

‚Ä¶and their imports and uses are stripped automatically.

</details>

---

<details>
  <summary><strong>Why are my CRUD routes missing?</strong></summary>

If you selected **‚ÄúNo CRUD‚Äù** during CLI setup, the example CRUD files were removed intentionally.
To restore them, either:

* Re-run the CLI and include CRUD, or
* Copy CRUD modules manually from the template folder.

</details>

<details>
  <summary><strong>Can I use TypeScript with NeatNode?</strong></summary>

Yes! NeatNode supports both **JavaScript and TypeScript** templates.

During the CLI prompts, select **TypeScript** when asked for your preferred language:

```bash

```

Available TypeScript templates:
- **Basic Express (TS)** ‚Äî minimal setup with strict typing
- **REST API (TS)** ‚Äî *(coming soon)*
- **Socket.IO (TS)** ‚Äî *(coming soon)*

## Configuration & Environment

<details>
  <summary><strong>Where do I configure environment variables?</strong></summary>

Each template includes a `.env.example` file.
Copy it and rename to `.env`, then adjust values like:

```bash
PORT=5000
DB_URL=mongodb://localhost:27017/mydb
```

Your app automatically loads these via **dotenv** in `server.js`.

</details>

---

<details>
  <summary><strong>How do I change the default port?</strong></summary>

Open `server.js` in your generated project and modify:

```js
const PORT = process.env.PORT || 5000;
```

Then update `.env` accordingly.

</details>

---

<details>
  <summary><strong>Can I use NeatNode inside an existing project?</strong></summary>

You can, but it‚Äôs **not recommended**.
The CLI assumes it‚Äôs creating a new folder.
If you must, use:

```bash
npx neatnode .
```

and it‚Äôll scaffold into the current directory.

</details>

## Troubleshooting

<details>
  <summary><strong>I see ‚ÄúFolder already exists‚Äù error</strong></summary>

NeatNode won‚Äôt overwrite existing directories to avoid data loss.
Either:

```bash
cd existing-folder && npx neatnode .
```

or create a new one:

```bash
npx neatnode my-new-app
```

</details>

---

<details>
  <summary><strong>removeCrudReferences didn‚Äôt clean my app.js file</strong></summary>

Make sure your `app.js` includes both comment markers:

```js
// ROUTE_IMPORTS_START
// ROUTE_IMPORTS_END
// ROUTE_USES_START
// ROUTE_USES_END
```

The regex-based remover depends on these comments.

</details>

---

<details>
  <summary><strong>Why does placeholder replacement not work everywhere?</strong></summary>

By default, only occurrences of **`project-name`** inside `package.json` are replaced.
Future updates will support:

```bash
{{projectName}}, {{author}}, {{version}}
```

for broader replacement coverage.

</details>

---

<details>
  <summary><strong>How do I update NeatNode?</strong></summary>

If globally installed:

```bash
npm update -g neatnode
```

Or if using via `npx`, it always fetches the latest published version.

</details>
